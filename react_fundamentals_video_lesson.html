Welcome to React
A beginner-friendly introduction to the world's most popular JavaScript UI library — what it is, why it exists, and how it thinks.
The Problem
Why did we need React?
  • Web UIs grew complex — keeping the DOM in sync with data was painful
  • Spaghetti JavaScript: manual DOM updates led to bugs and inconsistency
  • No standard way to reuse UI pieces across a project
  • Facebook needed to scale UI development — that's where React was born
Definition
React is a JavaScript Library

What React IS

A library for building user interfaces using reusable components. It handles the "View" in MVC.

What React is NOT

A full framework. It doesn't handle routing, data fetching, or state management on its own.

Maintained by Meta (Facebook) and a huge open-source community. Released 2013.
Core Concept
Everything is a Component
// A simple React component
function Greeting({ name }) {
  return (
    <div className="card">
      <h1>Hello, {name}!</h1>
      <p>Welcome to React.</p>
    </div>
  );
}
Components are reusable, self-contained pieces of UI. Write once, use anywhere.
How React Works
The Virtual DOM
  • React keeps a lightweight copy of the real DOM in memory
  • When state changes, React diffs old vs new Virtual DOM
  • Only the changed parts of the real DOM get updated — very fast!
  • You declare what the UI should look like — React figures out the updates
